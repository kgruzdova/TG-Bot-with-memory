# Telegram-бот с памятью

Telegram-бот на базе **aiogram 3.x** и **OpenAI API** с двумя типами памяти: короткой (история диалога) и долгой (документы через эмбеддинги).

## Структура проекта

```
.
├── bot.py                 # Объединённый бот (короткая + долгая память)
├── bot_short_memory.py    # Бот только с короткой памятью
├── bot_long_memory.py     # Бот только с долгой памятью
├── company.txt            # Пример документа для загрузки
├── .env.example           # Шаблон переменных окружения
├── requirements.txt       # Зависимости
└── memory/                # Хранилище эмбеддингов (создаётся автоматически)
```

## Как работают боты

### bot_short_memory.py — короткая память

Хранит последние 10 сообщений диалога в оперативной памяти для каждого пользователя. При каждом запросе отправляет историю + текущее сообщение в ChatCompletion.

- Память сбрасывается при перезапуске бота или по команде `/start`
- Подходит для обычного диалога с контекстом

### bot_long_memory.py — долгая память

Пользователь загружает документы (PDF, TXT, DOCX). Бот разбивает текст на чанки, строит эмбеддинги через OpenAI Embeddings и сохраняет на диск. При вопросе — ищет релевантные фрагменты и формирует ответ строго по документу.

- Память переживает перезапуск (хранится в `./memory/`)
- Подходит для Q&A по документам

### bot.py — двойная память (рекомендуется)

Объединяет оба подхода:

1. При получении текстового сообщения ищет релевантные фрагменты в загруженных документах (долгая память)
2. Подставляет найденный контекст + историю последних сообщений (короткая память)
3. Генерирует ответ через ChatCompletion

| Тип памяти | Хранилище | Что хранит | Время жизни |
|---|---|---|---|
| Короткая | `dict` в RAM | Последние 10 сообщений | До перезапуска / `/start` |
| Долгая | JSON на диске | Эмбеддинги документов | Постоянно |

## Быстрый старт

### 1. Клонировать и создать виртуальное окружение

```bash
python -m venv venv

# Windows (PowerShell)
.\venv\Scripts\Activate.ps1

# Linux / macOS
source venv/bin/activate
```

### 2. Установить зависимости

```bash
pip install -r requirements.txt
```

### 3. Настроить переменные окружения

```bash
cp .env.example .env
```

Отредактировать `.env`:

```env
BOT_TOKEN=ваш_токен_от_BotFather
OPENAI_API_KEY=ваш_ключ_OpenAI

# Опционально
CHAT_MODEL=gpt-4
EMBED_MODEL=text-embedding-3-small
```

### 4. Запустить

```bash
# Объединённый бот (рекомендуется)
python bot.py

# Или отдельно:
python bot_short_memory.py
python bot_long_memory.py
```

## Команды бота

| Команда | bot.py | bot_short | bot_long | Описание |
|---|---|---|---|---|
| `/start` | + | + | + | Сбросить диалог / приветствие |
| `/clear` | + | | + | Очистить базу документов |
| `/status` | + | | | Состояние обеих памятей |

## Загрузка документов

Поддерживаемые форматы: **PDF**, **TXT**, **DOCX**.

1. Отправьте файл боту в Telegram
2. Бот разобьёт текст на фрагменты (~500 символов)
3. Построит эмбеддинги через OpenAI Embeddings API
4. Сохранит в `./memory/user_{id}.json`
5. Теперь можно задавать вопросы по документу

## Переменные окружения

| Переменная | Обязательная | По умолчанию | Описание |
|---|---|---|---|
| `BOT_TOKEN` | да | — | Токен Telegram-бота |
| `OPENAI_API_KEY` | да | — | Ключ API OpenAI |
| `CHAT_MODEL` | нет | `gpt-4` | Модель для генерации ответов |
| `EMBED_MODEL` | нет | `text-embedding-3-small` | Модель для эмбеддингов |

## Технологии

- **Python 3.10+**
- **aiogram 3.x** — асинхронный фреймворк для Telegram Bot API
- **OpenAI API** — ChatCompletion + Embeddings
- **numpy** — косинусный поиск по эмбеддингам
- **pdfplumber** — извлечение текста из PDF
- **python-docx** — извлечение текста из DOCX
